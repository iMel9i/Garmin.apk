# Задачи разработки GHUD Lite

## Текущая цель: Улучшение настроек отладки и интеграции навигации

### Выполненные задачи
- [x] **Реализация UI отладки**: Создан `DebugActivity` для отображения данных отладки в реальном времени от OSM, парсинга навигации и вывода HUD.
- [x] **Создание настраиваемых параметров приложения**: Реализованы `AppConfigManager` and `AppSettingsActivity`, позволяющие пользователям управлять навигационными приложениями и правилами парсинга.
- [x] **Добавление UI настройки порога превышения скорости**: Интегрирован `SeekBar` в `MainActivity` для настройки `speeding_threshold`.
- [x] **Интеграция AppConfigManager**: Обновлен `NavigationNotificationListener` для использования динамических конфигураций из `AppConfigManager` для парсинга уведомлений.
- [x] **Исправление завершения сервиса/приложения**: Обеспечено, что кнопка "Выход" в уведомлении останавливает сервис и закрывает приложение (добавлен `setPackage` в broadcast intent).
- [x] **Обновление SDK**: Обновлены `compileSdkVersion` и `targetSdkVersion` до 35.

### В процессе
- [ ] **Сборка и установка**: Запуск `build_and_install.ps1` для проверки изменений на устройстве.

### Ожидающие / Следующие шаги
- [ ] **Проверка парсинга навигации Яндекс**: Проверить, работает ли общая логика парсинга корректно с динамической конфигурацией для Яндекс Карт/Навигатора.
- [ ] **Улучшение логики направлений HUD**: В настоящее время `HudService` использует жестко закодированный метод `parseDirection`. Возможно, его нужно перенести в `AppConfigManager` или сделать более гибким, если разные приложения используют сильно отличающиеся форматы инструкций.
- [ ] **Тестирование ограничений скорости и камер**: Проверить предупреждения об ограничениях скорости и камерах OSM в реальных условиях вождения.
